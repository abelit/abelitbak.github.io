Oracle网络服务相关概念

Oracle Net体系结构

Oracle Net 监听器是一个位于 Oracle Net 基础层之上的应用程序。数据库通过侦听器从客户端应用程序接收初始连接。

管理网络地址信息

使用本地化管理

配置文件通常创建在$ORACLE_HOME/network/admin目录和$ORACLE_BASE_HOME/network/admin只读 Oracle 主目录的目录中。但是，可以在其他目录中创建配置文件。Oracle Net 检查配置文件的其他目录。例如检查文件的顺序tnsnames.ora是：

  • 指定的目录TNS_ADMIN环境变量。
  • 如果TNS_ADMIN未设置环境变量或在目录中未找到该文件TNS_ADMIN，则 Oracle Net 会在目录中查找该文件ORACLE_HOME/network/admin。对于只读的 Oracle 主目录，Oracle Net 检查目录ORACLE_BASE_HOME/network/admin。
  • 对于只读的 Oracle 主目录，如果在目录中找不到文件ORACLE_BASE_HOME/network/admin，Oracle Net 将检查该ORACLE_HOME/network/admin目录。

笔记：

在 Microsoft Windows 上， TNS_ADMIN如果在进程的环境中设置了环境变量，则会使用它。如果TNS_ADMIN环境变量未在环境中定义，或者进程是没有环境的服务，则 Microsoft Windows 会扫描注册表以查找参数TNS_ADMIN 。

使用目录服务器进行集中管理

当 Oracle 网络使用目录服务器时，目录服务器将全局数据库链接作为网络中 Oracle 数据库的网络服务名称进行管理。用户和程序可以使用全局链接访问相应数据库中的对象。

Oracle Net Services 使用集中式目录服务器作为存储连接标识符的主要方法之一。客户端在其连接字符串中使用连接标识符，目录服务器将连接标识符解析为连接描述符，然后传递回客户端。此功能称为目录命名。这种类型的基础设施降低了管理和配置网络资源的成本。

Oracle Internet Directory 提供了一种用于管理和配置分布式 Oracle 网络的集中机制。目录服务器可以替换客户端和服务器端的本地化tnsnames.ora文件。

下图显示了客户端通过目录服务器解析连接标识符。

  • 客户端联系目录服务器以将连接标识符解析为连接描述符。

  • 目录服务器解析连接标识符并检索客户端的连接描述符。

  • 客户端使用连接描述符向侦听器发送连接请求。

数据库实例

一个数据库至少有一个实例。一个实例由称为系统全局区 (SGA) 的内存区域和 Oracle 后台进程组成。实例的内存和进程有效地管理关联数据库的数据并为数据库用户提供服务。实例由实例名称标识,实例名称默认为数据库实例的 Oracle 系统标识符 (SID)。

数据库服务

Oracle 数据库作为服务呈现给客户. 一个数据库可以有一个或多个与之关联的服务。服务名称是数据库的逻辑表示。当一个实例启动时，它会使用一个或多个服务名称向侦听器注册自己。当客户端程序或数据库连接到侦听器时，它会请求连接到服务。

一个服务名可以标识多个数据库实例，一个实例可以属于多个服务。为此，侦听器充当客户端和实例之间的中介，并将连接请求路由到适当的实例。连接到服务的客户端不需要指定他们需要哪个实例。

这服务名称由服务器参数文件中的 SERVICE_NAMES 初始化参数。服务器参数文件使您能够使用命令更改初始化参数ALTER SYSTEM，并在关闭和启动期间进行更改。DBMS_SERVICE 包也可用于创建服务。服务名称默认为全局数据库名称，该名称由数据库名称（DB_NAME 初始化参数）和域名（DB_DOMAIN 初始化参数）组成。

提示

SERVICE_NAMES从 Oracle Database 19c 开始，客户不再使用该参数。要管理您的服务，Oracle 建议您使用SRVCTL或GDSCTL命令行实用程序或DBMS_SERVICE包。

连接描述符

连接描述符配置

连接描述符由一个或多个侦听器的协议地址和文件中目标服务的连接信息组成tnsnames.ora。

sales= 
 (DESCRIPTION= 
  (ADDRESS=(PROTOCOL=tcp)(HOST=sales-server)(PORT=1521)) 
  (CONNECT_DATA= 
     (SID=sales) 
     (SERVICE_NAME=sales.us.example.com) 
     (INSTANCE_NAME=sales) ))

地址部分包含以下内容：

  • PROTOCOL 参数，标识侦听器协议地址。该协议tcp适用于 TCP/IP。

  • HOST参数，标识主机名。

  • PORT 参数，标识端口。端口为1521，默认端口号。

  • 可选的 HTTPS_PROXY 和 HTTPS_PROXY_PORT 参数，允许数据库客户端连接遍历组织的转发 Web 代理。这些参数仅适用于 PROTOCOL=TCPS 的连接描述符。

CONNECT_DATA 部分包含以下内容：

  • SID 参数，标识Oracle 数据库的系统标识符(SID)。

  • SERVICE_NAME 参数，标识服务。

此连接描述符参数的值来自初始化参数文件中的 SERVICE_NAMES 初始化参数（SERVICE_NAMES 使用结尾的 S）。SERVICE_NAMES 初始化参数通常是全局数据库名，包括数据库名和域名。

提示

笔记：SERVICE_NAMES从 Oracle Database 19c 开始，客户不再使用该参数。要管理您的服务，Oracle 建议您使用SRVCTL或GDSCTL命令行实用程序或DBMS_SERVICE包。

  • INSTANCE_NAME 参数，标识数据库实例。实例名称是可选的。

初始化参数文件中的 INSTANCE_NAME 参数默认为安装或创建数据库时输入的 SID。

多个监听器增强服务可访问性

(DESCRIPTION=
   (ADDRESS_LIST= 
    (ADDRESS=(PROTOCOL=tcp)(HOST=sales1-server)(PORT=1521) )
     (ADDRESS=(PROTOCOL=tcp)(HOST=sales2-server)(PORT=1521) )) 
  ( CONNECT_DATA= 
    (SERVICE_NAME=sales.us.example.com)))

多侦听器配置还使您能够单独或结合使用故障转移和负载平衡功能。

  • 连接时故障转移
  • 透明应用故障转移
  • 客户端负载均衡
  • 连接负载均衡

在共享服务器配置中，侦听器按以下顺序选择调度程序：

  • 负载最少的节点

  • 负载最少的实例

  • 该实例的负载最少的调度程序

在专用服务器配置中，侦听器按以下顺序选择实例：

  • 负载最少的节点

  • 负载最少的实例

如果数据库服务在多个节点上有多个实例，则侦听器会在负载最少的节点上选择负载最少的实例。

服务注册

指定实例名称

如果需要连接到数据库的特定实例，则客户端可以在连接描述符中指定特定实例的 INSTANCE_NAME。此功能对于 Oracle RAC 配置可能很有用。

(DESCRIPTION= 
  (ADDRESS=(PROTOCOL=tcp)(HOST=sales-server)(PORT=1521)) 
  (CONNECT_DATA= 
    (SERVICE_NAME=sales.us.example.com) (INSTANCE_NAME=sales1) ))

指定服务处理器

始终希望使用特定服务处理程序类型的客户端可以使用连接描述符来指定服务处理程序类型。在以下示例中，连接描述符用于(SERVER=shared)在连接到数据库时请求调度程序。默认情况下，数据库可能配置为使用专用服务器。

(DESCRIPTION= 
  (ADDRESS=(PROTOCOL=tcp)(HOST=sales-server)(PORT=1521)) 
  (CONNECT_DATA= 
    (SERVICE_NAME=sales.us.example.com) (SERVER=shared) ))

当侦听器收到客户端请求时，它会选择一个已注册的服务处理程序。根据所选处理程序的类型、使用的通信协议和数据库服务器的操作系统，侦听器执行以下操作之一：

  • 将连接请求直接交给调度员。

  • 将重定向消息发送回客户端，其中包含调度程序或专用服务器进程的位置。然后客户端直接连接到调度程序或专用服务器进程。

  • 生成专用服务器进程并将客户端连接传递给专用服务器进程。

监听器完成对客户端的连接操作后，客户端直接与Oracle数据库通信，无需监听器的参与。侦听器恢复侦听传入的网络会话。指定服务处理程序时应考虑以下因素：

  • 如果您希望客户端使用专用服务器，则指定(SERVER=dedicated). 如果未设置 SERVER 参数，则假定共享服务器配置。但是，如果没有可用的调度程序，客户端将使用专用服务器。

  • 如果服务器上启用了数据库驻留连接池，则指定(SERVER=pooled)从池中获取连接。如果服务器上未启用数据库驻留连接池，则客户端请求将被拒绝，并且用户会收到一条错误消息。

数据库常驻连接池

数据库驻留连接池在数据库服务器中提供了一个连接池，用于典型的 Web 应用程序使用场景，在这些场景中，应用程序获取数据库连接，在其上工作相对较短的时间，然后释放它。数据库常驻连接池“专用”服务器。池服务器相当于服务器前台进程和数据库会话的组合。数据库常驻连接池使用服务器和侦听器之间的动态注册。它不能使用静态注册。

数据库驻留连接池补充了在中间层进程中的线程之间共享连接的中间层连接池。此外，它还支持在同一中间层主机上什至跨中间层主机的中间层进程之间共享数据库连接。这导致支持大量客户端连接所需的关键数据库资源显着减少，从而减少数据库层内存占用并提高中间层和数据库层的可扩展性。拥有一个随时可用的服务器池还有一个额外的好处，即可以降低创建和关闭客户端连接的成本。

数据库驻留连接池为跨客户端应用程序和进程的专用连接。此功能对于必须保持与数据库的持久连接并优化服务器资源（如内存）的应用程序非常有用。

从数据库驻留连接池中获取连接的客户端持久连接到后台进程，连接代理，而不是专用服务器。连接代理实现池功能并执行从客户端到具有会话的专用服务器池的入站连接的多路复用。

当客户端必须执行数据库工作时，连接代理从池中选择一个专用服务器并将其分配给客户端。随后，客户端直接连接到专用服务器，直到请求得到服务。服务器处理完客户端请求后，服务器返回池中，客户端的连接恢复到连接代理。

专用服务器进程通过连接代理进程处理连接

命名方法

命名方法是客户端应用程序在尝试连接到数据库服务时用于将连接标识符解析为连接描述符的解析方法。用户通过提供连接字符串来发起连接请求。连接字符串包括用户名和密码，以及连接标识符。连接标识符可以是连接描述符或解析为连接描述符的名称。

选择命名方式

选择适当的命名方法来映射名称以连接描述符取决于组织的规模。

  • 对于只有几个数据库的小型组织，使用 Easy Connect 命名与数据库服务器的主机名建立 TCP/IP 连接，或使用本地命名将名称存储在tnsnames.ora客户端的文件中。

  • 对于拥有多个数据库的大型组织，使用目录命名将名称存储在中央目录服务器中。

  • 对于 Internet 网络，使用本地命名方法配置连接到数据库所需的应用程序 Web 服务器。

命名方法：优缺点

监听服务

oracle的监听服务主要涉及三个文件：服务端文件listener.ora、客户端文件tnsname.ora、sqlnet.ora。三者的默认位置都在：$ORACLE_HOME/network/admin/下。

  • tnsname.ora 配置文件是一个用于解析名称的客户端文件，服务器上通常有一个副本以便在服务器上运行客户端（sqlplus就是客户端）。
  • sqlnet.ora 配置文件是可选的，可能存在于服务端或客户端，它设置的各种默认值会影响所有客户机和服务器。
  • listener.ora 配置文件存在于Oracle数据库服务端，默认位置为：$ORACLE_HOME/network/admin/listener.ora。即Oracle服务端需要开启一个监听器来实时监听有哪些客户端来请求连接我服务端，这个监听器的信息配置对应的就是listener.ora文件，此文件中定义了Oracle服务端所需要监听的端口协议以及实例SID。

创建监听

netca -silent -responsefile ${ORACLE_HOME}/assistants/netca/netca.rsp -lisport 1521

静态监听

静态注册就是实例启动时读取listener.ora文件的配置，将实例和服务注册到监听程序。无论何时启动一个数据库，默认地都有两条信息注册到监听器中：数据库服务器对应的实例和服务。

在静态注册中，无论是否启动，都会向侦听器注册一个实例。当客户端请求到来时，监听器打开一个专用连接，然后服务器检查实例是否未启动，然后它给出错误消息“Oracle 不可用”。静态侦听器配置在位于下面的 listener.ora 中完成。

静态注册时，listener.ora中的GLOBAL_DBNAME向外提供服务名，listener.ora中的SID_NAME提供注册的实例名。

动态监听

在动态注册中，注册由 PMON根据init.ora中的instance_name,service_names两个参数将实例和服务动态注册到listener中，使用默认端口 1521 执行。一旦数据库实例启动，其 PMON 进程就会向关联的侦听器注册实例详细信息。动态注册不需要在 listener.ora 文件中进行任何手动配置。 如果我们希望我们的侦听器不使用默认端口，那么我们需要更改实例的 init.ora 或 spfile 中的 LOCAL_LISTENER 参数。

注册动态监听服务

  • 方法一

注册监听：

show parameter local_listener;

alter system set local_listener='(DESCRIPTION=(ADDRESS=(PROTOCOL=TCP)(HOST=testdb)(PORT=1521)))' scope=both;
alter system register;
  • 方法二

在$ORACLE_HOME/network/admin/tnsnames.ora添加如下内容：

MYTESTDB=
(DESCRIPTION =
    (ADDRESS_LIST =
        (ADDRESS = (PROTOCOL = TCP)(HOST = testdb)(PORT = 1521))
    )
    (CONNECT_DATA =
        (SERVICE_NAME = TESTDB)
    )
)

注册监听：

show parameter local_listener;

alter system set local_listener='MYTESTDB' scope=both;
alter system register;
