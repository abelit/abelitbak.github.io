介绍

Oracle逻辑存储结构

  • 表空间

表空间是数据库的逻辑划分，一个表空间只能属于一个数据库。表空间(tablespace)是最大的逻辑单位，对应一个或多个数据文件，通常由相关的段组成。表空间的大小是它所对应的数据文件大小的总和。所有的数据库对象都存放在指定的表空间中。

  • 段

一个段是分配给一个逻辑结构（一个表、一个索引或其他对象）的一组区，是数据库对象使用的空间的集合；段可以有表段、索引段、回滚段、临时段和高速缓存段等。

  • 区

数据库存储空间分配的一个逻辑单位，它由连续数据块所组成。第一个段是由一个或多个盘区组成。当一段中间所有空间已完全使用，Oracle为该段分配一个新的范围。

  • 块

Oralce管理数据文件中存储空间的单位，为数据库使用的I/O的最小单位，最小的逻辑部件，其大小可不同于操作系统的标准I/O块大小。数据块的大小由DB_BLOCK_SIZE参数确定。块尺寸是处理Oracle更新、选择、或者插入数据事务的最小单位。当用户从表中选择数据时，选择操作从数据库文件中以块为单位读取或者提取数据。例如Oracle块的大小为8kb，即使只想检索4kb的字符的名字，也必须读取含有这4个字符的整个8kb的块。

表空间管理

默认表空间

SELECT property_name,property_value FROM database_properties WHERE property_name='DEFAULT_PERMANENT_TABLESPACE';

创建表空间

CREATE TABLESPACE <tablespace_name> DATAFILE '<path_to_file>' SIZE 8M AUTOEXTEND ON NEXT 8192 MAXSIZE UNLIMITED;

删除表空间

DROP TABLESPACE <tablespace_name> INCLUDING CONTENTS AND DATAFILES;

增数据文件

ALTER TABLESPACE <tablespace_name> ADD DATAFILE '<path_to_file>' SIZE 8M AUTOEXTEND ON NEXT 8192 MAXSIZE UNLIMITED;

删除数据文件

ALTER TABLESPACE <tablespace_name> DROP DATAFILE '<path_to_file>';

设置自动扩展

ALTER DATABASE DATAFILE '<path_to_file>' AUTOEXTEND ON NEXT 8192 MAXSIZE UNLIMITED;

调整大小

ALTER DATABASE DATAFILE '<path_to_file>' RESIZE <size>;

移动命名

  • 离线数据文件
ALTER DATABASE DATAFILE '<path_to_file>' OFFLINE;
  • 重命名数据文件(操作系统层面)
mv <oldname> <newname>
  • 重命名数据文件(数据库层面)
ALTER DATABASE RENAME FILE <oldname> TO <newname>;
  • 数据文件联机
ALTER DATABASE DATAFILE '<path_to_file>' ONLINE;

临时表空间管理

默认临时表空间

SELECT property_name,property_value FROM database_properties WHERE property_name='DEFAULT_TEMP_TABLESPACE';

创建临时表空间

CREATE TEMPORARY TABLESPACE <tablespace_name> TEMPFILE '<path_to_file>' SIZE 8M AUTOEXTEND ON NEXT 8192 MAXSIZE UNLIMITED;

删除临时表空间

DROP TABLESPACE <tablespace_name> INCLUDING CONTENTS AND DATAFILES [CASCADE CONSTRAINTS];

新增数据文件

ALTER TABLESPACE <tablespace_name> ADD TEMPFILE '<path_to_file>' SIZE 8M AUTOEXTEND ON NEXT 8192 MAXSIZE UNLIMITED;

删除数据文件

ALTER TABLESPACE <tablespace_name> DROP TEMPFILE '<path_to_file>';

设置自动扩展

ALTER DATABASE TEMPFILE '<path_to_file>' AUTOEXTEND ON NEXT 8192 MAXSIZE UNLIMITED;

调整大小

ALTER DATABASE TEMPFILE '<path_to_file>' RESIZE <size>;

移动命名

  • 离线数据文件
ALTER DATABASE TEMPFILE '<path_to_file>' OFFLINE;
  • 重命名数据文件(操作系统层面)
mv <oldname> <newname>
  • 重命名数据文件(数据库层面)
ALTER DATABASE RENAME FILE <oldname> to <newname>;
  • 数据文件联机
ALTER DATABASE TEMPFILE '<path_to_file>' ONLINE;

临时表空间组

  • 创建指定组的临时表空间
CREATE TEMPORARY TABLESPACE <tablespace_name> TEMPFILE '<path_to_file>' SIZE 8M AUTOEXTEND ON MAXSIZE UNLIMITED TABLESPACE GROUP <group_name>;
  • 加入临时表空间组
ALTER TABLESPACE <tablespace_name> TABLESPACE GROUP <group_name>;
  • 从表空间组中移除
ALTER TABLESPACE <tablespace_name> TABLESPACE GROUP '';

切换临时表空间

ALTER DATABASE DEFAULT TEMPORARY TABLESPACE <tablespace_name>;

收缩临时表空间

ALTER TABLESPACE <tablespace_name> SHRINK SPACE KEEP 8G;
 
ALTER TABLESPACE <tablespace_name> SHRINK TEMPFILE '<path_to_file>';

大文件表空间

大文件表空间是具有单个但可能非常大（最多 4G 块）数据文件的表空间。

大文件表空间的好处如下：

  • 具有 8K 块的大文件表空间可以包含 32 TB 的数据文件。具有 32K 块的大文件表空间可以包含 128 TB 的数据文件。Oracle 数据库中的最大数据文件数是有限的（通常为 64K 文件）。因此，大文件表空间可以显着提高 Oracle 数据库的存储容量。

  • 大文件表空间可以减少数据库所需的数据文件数量。另一个好处是可以调整DB_FILES初始化参数和and语句MAXDATAFILES的参数，以减少数据文件信息所需的SGA空间量和控制文件的大小。 CREATE DATABASECREATE CONTROLFILE

  • 大文件表空间通过提供数据文件透明性来简化数据库管理。该语句的 SQL 语法ALTER TABLESPACE允许您对表空间执行操作，而不是对底层的单个数据文件执行操作。

创建大文件表空间

CREATE BIGFILE TABLESPACE <tablespace_name> DATAFILE '<path_to_data_file>' SIZE <initial_size> AOTOEXTEND ON NEXT 1G MAXSIZE 1T;
  • 只有具有自动段空间管理的本地管理的表空间支持大文件表空间。
  • 大文件表空间旨在与自动存储管理 (Oracle ASM) 或其他支持条带化或 RAID 的逻辑卷管理器以及动态可扩展的逻辑卷一起使用。
  • 避免在不支持条带化的系统上创建大文件表空间，因为这会对并行查询执行和 RMAN 备份并行化产生负面影响。
  • 不建议在不支持大文件大小的平台上使用大文件表空间，这会限制表空间容量。

表空间查询

表空间使用率

常用
SET LINESIZE 300
COL tablespace FORMAT a20

-- permanent
SELECT
    df.tablespace_name AS tablespace,
    df.type            AS type,
    nvl(
        SUM(df.bytes), 0
    )                  AS file_bytes,
    nvl(
        SUM(df.max_bytes), 0
    )                  AS max_bytes,
    nvl(
        SUM(f.free), 0
    )                  AS free_bytes,
    nvl(
        SUM(df.max_bytes), 0
    ) - decode(
        df.type, 'UNDO', MAX(due.max_bytes), SUM(df.bytes) - SUM(f.free)
    )                  AS max_free_bytes,
    decode(
        df.type, 'UNDO', MAX(due.max_bytes), SUM(df.bytes) - SUM(f.free)
    )                  AS used_bytes,
    round(
        decode(
            SUM(df.max_bytes), 0, 0,(SUM(df.bytes) / SUM(df.max_bytes) * 100)
        ), 2
    )                  AS file_max_pct,
    round(
        decode(
            SUM(df.bytes), 0, 0,(decode(
                df.type, 'UNDO', MAX(due.max_bytes), SUM(df.bytes) - SUM(f.free)
            )) / SUM(df.bytes) * 100
        ), 2
    )                  AS used_file_pct,
    round(
        decode(
            SUM(df.bytes), 0, 0,(decode(
                df.type, 'UNDO', MAX(due.max_bytes), SUM(df.bytes) - SUM(f.free)
            )) / SUM(df.max_bytes) * 100
        ), 2
    )                  AS used_max_pct,
    COUNT(1)           AS file_num,
    decode(
        df.status, 'ONLINE', 1, 'OFFLINE', 2, 'READ ONLY', 3, 0
    )                  AS status
FROM
    (
        SELECT
            ddf.file_id,
            dt.contents AS type,
            dt.status,
            ddf.file_name,
            ddf.tablespace_name,
            trunc(
                ddf.bytes
            )           AS bytes,
            trunc(
                greatest(
                    ddf.bytes, ddf.maxbytes
                )
            )           AS max_bytes
        FROM
            dba_data_files  ddf,
            dba_tablespaces dt
        WHERE
            ddf.tablespace_name = dt.tablespace_name
    ) df,
    (
        SELECT
            trunc(
                SUM(bytes)
            ) AS free,
            file_id
        FROM
            dba_free_space
        GROUP BY
            file_id
    ) f,
    (
        SELECT
            tablespace_name,
            SUM(bytes) AS max_bytes
        FROM
            dba_undo_extents
        WHERE
            1 = 1
            AND status = 'UNEXPIRED'
        GROUP BY
            tablespace_name
    ) due
WHERE
    df.file_id = f.file_id (+)
    AND df.tablespace_name = due.tablespace_name (+)
GROUP BY
    df.tablespace_name,
    df.type,
    df.status
UNION ALL

SELECT
    y.name   AS tablespace,
    y.type   AS type,
    nvl(
        SUM(y.bytes), 0
    )        AS file_bytes,
    nvl(
        SUM(y.max_bytes), 0
    )        AS max_bytes,
    nvl(
        MAX(nvl(
            y.free_bytes, 0
        )), 0
    )        AS free,
    nvl(
        SUM(y.max_bytes), 0
    ) - nvl(
        SUM(y.bytes) - MAX(y.free_bytes), 0
    )        AS max_free_bytes,
    nvl(
        SUM(y.bytes) - MAX(y.free_bytes), 0
    )        AS used_bytes,
    round(
        decode(
            SUM(y.max_bytes), 0, 0,(SUM(y.bytes) / SUM(y.max_bytes) * 100)
        ), 2
    )        AS file_max_pct,
    round(
        decode(
            SUM(y.bytes), 0, 0,(SUM(y.bytes) - MAX(y.free_bytes)) / SUM(y.bytes) * 100
        ), 2
    )        AS used_file_pct,
    round(
        (nvl(
            SUM(y.bytes) - MAX(y.free_bytes), 0
        ) / SUM(y.max_bytes) * 100), 2
    )        AS used_max_pct,
    COUNT(1) AS file_num,
    decode(
        y.tbs_status, 'ONLINE', 1, 'OFFLINE', 2, 'READ ONLY', 3, 0
    )        AS status
FROM
    (
        SELECT
            dtf.tablespace_name AS name,
            dt.contents         AS type,
            dt.status           AS tbs_status,
            dtf.status          AS status,
            dtf.bytes           AS bytes,
            nvl(
                (
                    SELECT
                        ((f.total_blocks - nvl(
                            s.tot_used_blocks, 0
                        )) * vp.value)
                    FROM
                        (
                            SELECT
                                tablespace_name, SUM(used_blocks) AS tot_used_blocks
                            FROM
                                gv$sort_segment
                            WHERE
                                tablespace_name != 'DUMMY'
                            GROUP BY
                                tablespace_name
                        ) s,(
                            SELECT
                                tablespace_name, SUM(blocks) AS total_blocks
                            FROM
                                dba_temp_files
                            WHERE
                                tablespace_name != 'DUMMY'
                            GROUP BY
                                tablespace_name
                        ) f,(
                            SELECT
                                value
                            FROM
                                v$parameter
                            WHERE
                                name = 'db_block_size'
                        ) vp
                    WHERE
                        f.tablespace_name = s.tablespace_name(+)
                        AND f.tablespace_name = dtf.tablespace_name
                ), 0
            )                   AS free_bytes,
            CASE
            WHEN dtf.maxbytes = 0 THEN
            dtf.bytes
            ELSE
            dtf.maxbytes
            END                 AS max_bytes
        FROM
            sys.dba_temp_files  dtf,
            sys.dba_tablespaces dt
        WHERE
            dtf.tablespace_name = dt.tablespace_name
    ) y
GROUP BY
    y.name,
    y.type,
    y.tbs_status;

SELECT
    upper(dfs.tablespace_name)                                                                                   tbs,
    ddf.filenum                                                                                                  filenum,
    dt.contents                                                                                                  tbstype,
    ddf.toatal_bytes                                                                                             totalsz,
    ddf.toatal_bytes - dfs.total_free_bytes                                                                      usedsz,
    dfs.total_free_bytes                                                                                         freesz,
    round((ddf.toatal_bytes - dfs.total_free_bytes) * 100 / decode(ddf.toatal_bytes, 0, 1, ddf.toatal_bytes), 2) AS usedpct,
    ddf.max_bytes                                                                                                maxsz,
    ddf.max_bytes - ( ddf.toatal_bytes - dfs.total_free_bytes )                                                  maxfreesz,
    round((ddf.toatal_bytes - dfs.total_free_bytes) * 100 / ddf.max_bytes, 2)                                    AS maxusedpct,
    decode(dt.status, 'ONLINE', 1, 'OFFLINE', 2,
           'READ ONLY', 3, 0)                                                                                    AS status
FROM
    (
        SELECT
            tablespace_name,
            SUM(bytes) total_free_bytes
        FROM
            sys.dba_free_space
        GROUP BY
            tablespace_name
        UNION
        SELECT
            tablespace_name,
            SUM(free_space) total_free_bytes
        FROM
            sys.dba_temp_free_space
        GROUP BY
            tablespace_name
    )                   dfs,
    (
        SELECT
            tablespace_name,
            SUM(bytes) toatal_bytes,
            SUM(
                CASE autoextensible
                    WHEN 'YES' THEN
                        maxbytes
                    ELSE
                        bytes
                END
            )          max_bytes,
            COUNT(1)   filenum
        FROM
            sys.dba_data_files
        GROUP BY
            tablespace_name
        UNION
        SELECT
            tablespace_name,
            SUM(bytes) toatal_bytes,
            SUM(
                CASE autoextensible
                    WHEN 'YES' THEN
                        maxbytes
                    ELSE
                        bytes
                END
            )          max_bytes,
            COUNT(1)   filenum
        FROM
            sys.dba_temp_files
        GROUP BY
            tablespace_name
    )                   ddf,
    sys.dba_tablespaces dt
WHERE
        ddf.tablespace_name = dfs.tablespace_name
    AND ddf.tablespace_name = dt.tablespace_name
ORDER BY
    tbs;

            sys.dba_tablespaces dt
        WHERE
            dtf.tablespace_name = dt.tablespace_name
    ) y
GROUP BY
    y.name,
    y.type,
    y.tbs_status;

表空间数据增长

SELECT
    to_char(
        sp.begin_interval_time, 'YYYY-MM-DD'
    )         history_date,
    ts.tsname tablespace_name,
    MAX(round(
        (tsu.tablespace_size * dt.block_size) /(1024 * 1024), 2
    ))        current_size,
    MAX(round(
        (tsu.tablespace_usedsize * dt.block_size) /(1024 * 1024), 2
    ))        used_size
FROM
    dba_hist_tbspc_space_usage tsu,
    dba_hist_tablespace_stat   ts,
    dba_hist_snapshot          sp,
    dba_tablespaces            dt
WHERE
    tsu.tablespace_id = ts.ts#
    AND tsu.snap_id = sp.snap_id
    AND ts.tsname = dt.tablespace_name
GROUP BY
    to_char(
        sp.begin_interval_time, 'YYYY-MM-DD'
    ),
    ts.tsname
ORDER BY
    ts.tsname,
    history_date;

临时表空间数据文件

COL TEMP_FILE FOR A60;

SELECT
    f.tablespace_name,
    d.file_name AS "TEMP_FILE",
    round(
        (f.bytes_free + f.bytes_used) / 1024 / 1024 / 1024, 2
    )           AS total,
    round(
        nvl(
            p.bytes_used, 0
        ) / 1024 / 1024 / 1024, 2
    )           AS used,
    round(
        ((f.bytes_free + f.bytes_used) - nvl(
            p.bytes_used, 0
        )) / 1024 / 1024 / 1024, 2
    )           AS free
FROM
    sys.v_$temp_space_header f,
    dba_temp_files           d,
    sys.v_$temp_extent_pool  p
WHERE
    f.tablespace_name (+) = d.tablespace_name
    AND f.file_id (+) = d.file_id
    AND p.file_id (+) = d.file_id;

临时表空间使用会话与SQL

SELECT
    se.username,
    se.sid,
    su.extents,
    su.blocks * TO_NUMBER(rtrim(
        p.value
    )) AS space,
    tablespace,
    segtype,
    sql_text
FROM
    v$sort_usage su,
    v$parameter  p,
    v$session    se,
    v$sql        s
WHERE
    p.name = 'db_block_size'
    AND su.session_addr = se.saddr
    AND s.hash_value = su.sqlhash
    AND s.address = su.sqladdr
ORDER BY
    se.username,
    se.sid;

Undo表空间管理

什么是UNDO？

Oracle 数据库创建和管理用于回滚或撤消对数据库的更改的信息。此类信息包括事务操作的记录，主要是在它们被提交之前。这些记录统称为undo。

UNDO记录用于：

  • ROLLBACK发出语句 时回滚事务
  • 恢复数据库
  • 提供读取一致性
  • 使用 Oracle 闪回查询分析较早时间点的数据
  • 使用 Oracle 闪回功能从逻辑损坏中恢复

发出ROLLBACK语句时，撤消记录用于撤消未提交事务对数据库所做的更改。在数据库恢复期间，撤消记录用于撤消从重做日志应用到数据文件的任何未提交的更改。撤消记录通过在另一个用户更改数据的同时为正在访问数据的用户维护数据的前映像来提供读取一致性。

UNDO自动管理

  • UNDO实例参数
参数指令
show parameter undo
v$parameter
SELECT * FROM v$parameter WHERE name like 'undo%';
  • undo_management: UNDO管理方式，如果AUTO或 null，则启用自动撤消管理。如果MANUAL，设置手动撤消管理模式。默认值为AUTO。
  • undo_tablespace: 数据指定的UNDO表空间。
  • undo_retention: UNDO保留时间（单位为s），默认900s。

创建UNDO表空间

CREATE UNDO TABLESPACE <tablespace_name> DATAFILE '<path_to_file>' SIZE 8M REUSE AUTOEXTEND ON MAXSIZE UNLIMITED;

切换UNDO表空间

ALTER SYSTEM SET UNDO_TABLESPACE = <tablespace_name> SCOPE=BOTH;

查询UNDO Period

SELECT
    to_char(
        begin_time, 'YYYY-MM-DD HH24:MI'
    ) begin_time,
    to_char(
        end_time, 'YYYY-MM-DD HH24:MI'

UNDO相关数据字典

视图描述

表空间规划

分离活动表与静态表

静态表可能永远不需要更改或移动，活动表则可能需要频繁地管理、移动或重组。为了简化静态表的管理，可以将他们分离在一个专门的表空间中。

分离索引和表

索引可以与表分开管理--可以在保持基表不改变的同时创建或删除索引。如果使用ASM，负载均衡是自动的，可以将表及其相关索引放在同一个表空间中。

分离大对象与小对象

Oracle为小型表提供了优化选项（如缓存），但是不适用于大型表（大型表有自己的一套优化选项）。

应用程序表和核心表分开

Oracle核心对象存储在默认表空间中（SYSTEM、SYSAUX、TEMPORARY、UNDO）。不要在这些表空间中或在Oracle提供的任何用户下创建任何应用程序对象。
