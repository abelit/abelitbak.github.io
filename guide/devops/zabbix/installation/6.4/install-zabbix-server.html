基于CentOS环境

安装Zabbix依赖

CentOS 7.x
rpm -Uvh https://repo.zabbix.com/zabbix/6.4/rhel/7/x86_64/zabbix-release-6.4-1.el7.noarch.rpm
yum clean all
CentOS 8.x

支持版本CentOS 8.x Alma Linux 8 Rocky Linux 8

rpm -Uvh https://repo.zabbix.com/zabbix/6.4/rhel/8/x86_64/zabbix-release-6.4-1.el8.noarch.rpm
dnf clean all

切换PHP版本

dnf module switch-to php:7.4
CentOS 9.x

支持版本CentOS 9.x Alma Linux 9 Rocky Linux 9 编辑/etc/yum.repos.d/epel.repo,添加如下信息（禁用epel提供的zabbix源）：

[epel]
...
excludepkgs=zabbix*

安装Zabbix官方依赖：

rpm -Uvh https://repo.zabbix.com/zabbix/6.4/rhel/9/x86_64/zabbix-release-6.4-1.el9.noarch.rpm
dnf clean all

安装Zabbix Server,Frontend,Agent

CentOS 6.x-7.x

MySQL+Nginx
yum install zabbix-server-mysql zabbix-web-mysql zabbix-nginx-conf zabbix-sql-scripts zabbix-selinux-policy zabbix-agent
PG+Nginx
yum install zabbix-server-pgsql zabbix-web-pgsql zabbix-nginx-conf zabbix-sql-scripts zabbix-selinux-policy zabbix-agent

CentOS 8.x+

主要版本：

  • CentOS 8 Stream
  • CentOS 9 Stream
  • Alma Linux 8
  • Alma Linux 9
  • Rocky Linux 8
  • Rocky Linux 9
MySQL+Nginx
dnf install zabbix-server-mysql zabbix-web-mysql zabbix-nginx-conf zabbix-sql-scripts zabbix-selinux-policy zabbix-agent
PG+Nginx
dnf install zabbix-server-pgsql zabbix-web-pgsql zabbix-nginx-conf zabbix-sql-scripts zabbix-selinux-policy zabbix-agent

创建初始数据库

MySQL

  • 创建数据库
create database zabbix character set utf8mb4 collate utf8mb4_bin;
create user zabbix@localhost identified by 'Password@123';
grant all privileges on zabbix.* to zabbix@localhost;
set global log_bin_trust_function_creators = 1;
quit;
  • 导入数据结构
zcat /usr/share/zabbix-sql-scripts/mysql/server.sql.gz | mysql --default-character-set=utf8mb4 -uzabbix -p zabbix
  • 还原数据设置
set global log_bin_trust_function_creators = 0;

PostgreSQL

  • 创建数据库
sudo -u postgres createuser --pwprompt zabbix
sudo -u postgres createdb -O zabbix zabbix
  • 导入数据结构
zcat /usr/share/zabbix-sql-scripts/postgresql/server.sql.gz | sudo -u zabbix psql zabbix

配置Zabbix Server

配置Zabbix Server连接数据库信息

编辑/etc/zabbix/zabbix_server.conf,修改如下信息：

DBPassword=Password@123

配置 Zabbix Frontend PHP信息

编辑/etc/nginx/conf.d/zabbix.conf，修改如下信息：

listen 8080;
server_name example.com;

启动Zabbix Server

systemctl restart zabbix-server zabbix-agent nginx php-fpm
systemctl enable zabbix-server zabbix-agent nginx php-fpm

Zabbix Oracle by ODBC配置

Zabbix Oracle by ODBC Server端配置

  • 下载Oracle驱动
wget https://download.oracle.com/otn_software/linux/instantclient/216000/instantclient-odbc-linux.x64-21.6.0.0.0dbru.zip
  • 如果是zabbix server，则在文件 /etc/sysconfig/zabbix-server中添加以下内容, 如果是zabbix，在文件 /etc/sysconfig/zabbix-proxy中添加以下内容。
cat > /etc/sysconfig/zabbix-server <<-EOF
ORACLE_HOME=/opt/instantclient_21_6
PATH=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/opt/instantclient_21_6
LD_LIBRARY_PATH=/opt/instantclient_21_6:/usr/lib64
TNS_ADMIN=/opt/instantclient_21_6/network/admin

export ORACLE_HOME
export PATH
export LD_LIBRARY_PATH
export TNS_ADMIN
EOF
  • 安装ODBC
yum install unixODBC-devel unixODBC
  • ODBC驱动配置

Zabbix Oracle by ODBC Client端配置

  • 在数据库中创建用户并授权访问相应视图
CREATE USER yszabbix IDENTIFIED BY YsZabbix2022_ZBX;
GRANT SELECT ANY TABLE TO yszabbix;
GRANT CREATE SESSION TO yszabbix;
GRANT SELECT ANY DICTIONARY TO yszabbix;
GRANT UNLIMITED TABLESPACE TO yszabbix;
GRANT SELECT ON V_$SESSION TO yszabbix;
GRANT SELECT ON V_$SYSTEM_EVENT TO yszabbix;
GRANT SELECT ON V_$EVENT_NAME TO yszabbix;
GRANT SELECT ON V_$RECOVERY_FILE_DEST TO yszabbix;
  • 模板中使用参数
Oracle: Service's TCP port state

net.tcp.service[tcp,{HOST.CONN},{$ORACLE.PORT}]

net.tcp.service[tcp,,{$ORACLE.PORT}]



Tablespace discovery

Oracle TBS 'H1_HIS_DATA': Tablespace allocated, percent

oracle.tbs_used_pct["H1_HIS_DATA"]

Allocated bytes/Max bytes*100


select * from v$event_name;

select * from v$eventmetric;

SELECT
    t2.name,
    to_char(t1.begin_time,'yyyy-mm-dd hh24:mi:ss'),
    to_char(t1.end_time,'yyyy-mm-dd hh24:mi:ss'),
    t1.num_sess_waiting,
    t1.time_waited,
    wait_count,
     t1.time_waited/t1.wait_count
FROM
         v$eventmetric t1
    JOIN v$event_name t2 ON t1.event_id = t2.event_id
    where t1.wait_count>0
    order by  t1.time_waited desc;
